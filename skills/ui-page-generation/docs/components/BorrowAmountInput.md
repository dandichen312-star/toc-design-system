# BorrowAmountInput（可借金额输入）

## 1. 用途与边界

BorrowAmountInput 是借款页专用金额输入组件，用于承载“可借金额”的输入、修改、校验和快速回填。

BorrowAmountInput 负责：

- 展示币种符号。
- 展示和编辑借款金额。
- 展示固定存在的限额提示。
- 在输入中提供删除能力。
- 在需要时提供 `全部借出` 快捷操作。
- 在输入中实时校验金额并展示报错。

BorrowAmountInput 不负责：

- 不替代通用 `Input` 组件。
- 不负责页面级表单分组。
- 不负责协议、按钮、提交结果或页面流程跳转。
- 第一版不包含小 `i` 提示和说明气泡。
- 第一版不支持只读态。
- 第一版不支持禁用态。

前置业务约束：

- `用户当前可借额度上限 >= 单笔最高可借额度上限`
- 因此组件默认填充 `单笔最高可借额度上限` 时，不会出现默认值超过当前可借额度的冲突。

## 2. 结构

```txt
BorrowAmountInput
├── currency-symbol
├── amount-value
├── clear
├── all-borrow
├── limit-tip
└── error-message
```

规则：

- 组件基础结构固定为：`currency-symbol + amount-value + limit-tip + clear + all-borrow`。
- 第一版移除 `info-tip`，不显示小 `i`，也不显示对应气泡说明。
- `currency-symbol` 固定显示 `¥`。
- `amount-value` 只允许纯数字输入，不带千分位。
- `limit-tip` 固定存在，不因状态切换而消失。
- `limit-tip` 固定放在金额输入线的上方。
- `limit-tip` 与金额输入区左对齐。
- `error-message` 只在命中报错态时出现，位置在 `limit-tip` 下方。

## 3. 状态

```txt
filled:
  含义：输入完成态。
  默认：页面初始自动填充值即进入 filled。

editing:
  含义：用户正在输入。

empty-editing:
  含义：用户已进入输入中，但当前值被清空。

error:
  含义：用户输入中实时校验不通过。
```

规则：

- 页面初始进入时，默认自动填充 `单笔最高可借额度上限`。
- 页面初始自动填充值直接算 `filled`。
- 页面初始 `filled` 态下，不显示 `clear`。
- 页面初始 `filled` 态下，不显示 `全部借出`。
- 从 `filled` 进入 `editing`：点击金额区域任意位置即可进入编辑。
- `empty-editing` 单独定义，不合并进普通 `editing`。
- `empty-editing` 状态下，金额主显示区只保留 `¥` 和输入光标，不显示占位文案。
- 用户结束输入且金额合法后，组件回到 `filled`。
- 正常态和输入中态不区分输入线样式；只有 `error` 使用红色输入线。

## 4. 交互

### 4.1 默认值与回填

规则：

- 默认自动填充 `单笔最高可借额度上限`。
- `全部借出` 点击后，金额直接填充为 `单笔最高可借额度上限`。

### 4.2 删除与编辑

规则：

- 删除按钮只在输入中出现。
- 点击删除按钮后，金额清空为无值，但组件仍保持 `empty-editing`。
- 删除后允许继续输入，不回填默认值。
- 删除后 `全部借出` 要出现，允许用户一键回填最大可借金额。

### 4.3 全部借出

规则：

- `全部借出` 默认不是初始必显。
- 页面初始默认填充值为最大可借金额时，`全部借出` 不显示。
- 用户修改过金额后，`全部借出` 显示。
- `error` 状态下，`全部借出` 继续显示。

### 4.4 输入约束

规则：

- 第一版金额只允许输入整数。
- 非数字字符在输入阶段就拦截，不允许进入值区。
- 前导零允许输入，但最终按正常整数规则处理。

## 5. 校验与报错

### 5.1 校验时机

规则：

- 第一版使用输入中实时校验。
- 用户输入中一旦不满足条件，立即进入 `error`。
- 用户继续修改，满足条件后，立即回到正常态。

### 5.2 报错表现

规则：

- `error` 状态下，输入线变为红色。
- 在输入下方展示红色报错文案。
- `limit-tip` 不隐藏，继续保留。
- `clear` 和 `全部借出` 在 `error` 状态下继续显示。

### 5.3 第一版校验项

```txt
1. 借款金额必须为 100 的整数倍
2. 单笔借款金额不得低于 500 元
3. 已超出当前最高可借金额 {dynamic-max} 元
```

规则：

- 报错文案中的额度值使用真实动态额度值替换，不写死示例值。
- 超上限报错默认使用短文案。
- 只有命中“可分多笔借”的特定业务条件时，才升级为带补充说明的长文案。

报错文案：

```txt
100 的整数倍：
  借款金额必须为100的整数倍

最小值：
  单笔借款金额不得低于500元

超上限默认短文案：
  已超出单笔最高可借金额{dynamic-max}元

超上限长文案：
  已超出单笔最高可借金额{dynamic-max}元，您的当前额度是{current-limit}元，可分多笔进行借款
```

## 6. AI 使用规则

- BorrowAmountInput 是借款页专用金额输入组件。
- 借款页的金额输入必须优先使用 `BorrowAmountInput`，不能用普通 `Input` 替代。
- AI 不允许自行用 `Input + Button + Toast + 文案` 临时拼装成 BorrowAmountInput。
- 页面存在“可借金额输入”时，优先识别为 `BorrowAmountInput`，而不是通用 `Input amount`。
- 第一版只允许使用 `filled / editing / empty-editing / error` 四种状态。
- 第一版只允许使用当前已确认的三类校验，不扩展更多业务校验项。
- 第一版不生成小 `i` 提示和对应气泡。
- 第一版不生成只读态和禁用态。

  color-text-primary

  text-body-sm

  text-body-sm
  color-text-danger

  color-border-danger

正常输入线：
  沿用当前 Input 默认输入线规则

删除图标：
  Icon.md / clear

全部借出：
  复用现有 Button / text-button-xs 对应的小型弱操作规则

规则：

- BorrowAmountInput 相关的字色、字号、间距、圆角必须优先来自 `DESIGN_TOKENS.md`。
- 未经确认，不新增新的输入态、按钮态或图标态。
- 没有明确确认的视觉值，不在本组件文档中补写。
