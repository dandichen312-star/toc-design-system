# 需求梳理示例

## 示例 1：模糊输入 → 澄清 → PRD 摘要

### 用户输入

> 做一个绑卡页面

### Agent 理解复述

您需要的是银行卡绑定页，属于资料采集流程中的一环。我先确认几个关键点。

### 澄清问题（第 1 轮）

1. 绑卡成功后跳转到哪里？（继续借款申请 / 返回上一页 / 指定页面）
2. 表单需要哪些字段？（卡号、手机号、验证码、持卡人姓名？）
3. 提交前是否需要勾选《银行卡服务协议》？

### 用户回答

> 绑完继续填借款资料；要卡号+手机号+验证码；要勾选协议

### PRD 核心输出（节选）

```markdown
## 3. 页面清单

| # | 页面名称 | Page Pattern | 说明 |
|---|----------|--------------|------|
| 1 | 绑定银行卡 | form-flow-page | 借款资料流程第 2 步 |

## 4.1 绑定银行卡

**内容模块**

| 序号 | 模块 | 组件映射 | 说明 |
|------|------|----------|------|
| 1 | 绑卡表单 | Form + FormItem + Input | 卡号、手机号、验证码 |
| 2 | 协议 | Agreement | 勾选《银行卡服务协议》 |

**action-footer：** 主按钮「下一步」，协议未勾选时 disabled

## 10. UI 生成交接摘要

- Pattern: `form-flow-page`
- 必读: Form.md, Input.md, Agreement.md, Button.md
- 成功后跳转: 借款资料页（URL/路由待开发确认）
```

## 示例 2：较完整输入 → 少澄清

### 用户输入

> 借款申请结果页：成功展示借款金额 5000 元、期限 12 期、年化 7.2%，主按钮「完成」回首页，次按钮「查看详情」进借款详情。失败态展示失败原因和「重新申请」。

### Agent 行为

- P0 信息已齐，仅确认：审核中态是否需要？→ 用户说不需要
- 直接输出完整 PRD，保存为 `.cursor/requirements/loan-apply-result-prd.md`

### 映射要点

- Page Pattern: `result-page`
- 成功态金额/利率: `AmountText`
- 两种结果态 → 状态矩阵 2 行
- 组件: Header, Card, Button, AmountText, Icon(status)

## 示例 3：多页面流程

### 用户输入

> 做一个优惠券列表，点进去看详情，可以领取和使用

### Agent 澄清（节选）

1. 列表页是否需要 Tabs（可用/已用/过期）？
2. 详情页「领取」与「去使用」是同一页还是领取后变状态？
3. 领取失败如何提示？

### 输出结构

- 页面 1: 优惠券列表 → `list-page`
- 页面 2: 优惠券详情 → `detail-page`
- 附 mermaid 跳转图
- 领取成功: Toast + 按钮态变为「去使用」

## 示例 4：借款首页 — 两轮澄清（标准流程）

### 用户输入

> 我想设计一个 H5 页面

### 第 1 轮（路由）

AskQuestion：页面类型=借款首页；场景=老客已激活；CTA=去借款；产出=先 PRD 再 HTML。

### Agent 行为（第 1 轮后）

**不得写 PRD。** 进入第 2 轮。

### 第 2 轮（结构，必问）

**Q1（多选）** 借款首页包含哪些内容模块？

- 额度主卡、任务卡、权益卡、活动卡、产品推荐、Header 消息入口、bottom-tab…

**Q2（若选了流程起点）** 除首页外，借款流程还包括哪些页面？

- 借款申请页、成本弹窗、等待页、结果页…

### 用户回答

> 只要额度主卡 + bottom-tab；流程后续有借款页、等待页、结果页

### 之后才输出 PRD

§3 含 4 页清单；§4.1 模块表仅额度主卡 + Tab；§8 组件：Card、AmountText、Button、Header、Icon(tab)。

## 反例：Agent 不应做的事

**用户：** 帮我梳理一下首页需求

**错误 ❌**  
第 1 轮问完页面类型/场景/CTA 后直接生成 PRD 或 HTML，未问「首页包含哪些模块」。

**正确 ✅**  
第 2 轮用 HA01 多选确认模块，再输出 PRD §4 模块表，并询问是否进入 UI 生成。
