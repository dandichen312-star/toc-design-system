# 澄清问题库

按场景组织。Agent 根据 Step 2 缺口从中选题，不要整库抛出。

## 通用（所有页面）

| ID | 优先级 | 问题 | 默认假设 |
|----|--------|------|----------|
| G01 | P0 | 这个页面/功能的核心目标是什么？用户完成后应达到什么结果？ | — |
| G02 | P0 | 用户从哪个入口进入？成功后跳转到哪里？ | 返回上一页 |
| G03 | P0 | 是否需要登录态？未登录时如何处理？ | 需登录，跳转登录 |
| G04 | P1 | 页面标题栏文案是什么？是否需要返回键？ | 有返回，标题由功能名推导 |
| G05 | P2 | 是否需要顶部全局通知条（Notice）？ | **无**（仅用户提及或业务强依赖时才问） |
| G06 | P1 | 是否需要底部固定主按钮（action-footer）？ | 有主操作时默认需要 |
| G07 | P2 | 是否需要特殊背景（渐变/插图）？ | 使用默认 page 背景 |

## 条件分支（第 2 轮 P0，Step 3 必用）

第 1 轮路由题答完后**必须**进入本节；答完适用分支后才允许写 PRD。

### BR-A 主页面内容模块（home-page / account-asset-page 等）

| ID | 优先级 | 问题 | 默认假设 |
|----|--------|------|----------|

**借款首页 `loan-home-page` 多选选项**（`AskQuestion`，`allow_multiple: true`）：

| 选项 ID | 模块 | 组件映射 |
|---------|------|----------|
| hero-amount | 额度主卡 | Card + AmountText + Button |

- 借款首页：**额度主卡 + bottom-tab 为规范必选**（`PAGE_RULES` loan-home-page）；其余由用户多选。

- 选了 `hero-amount` 时，可追加 H02：额度主卡展示哪些字段。

**任务首页 `task-home-page` 多选：** 状态卡、任务列表卡、bottom-tab（必选）。

## 使用规则

1. 同一轮澄清中，同一页面最多选 3 个场景块的问题。
2. 用户已回答的内容不再重复提问。
3. 默认假设写入 PRD §9 待确认项，除非用户明确确认。
4. **Notice 默认无**：不要每单都问 G05；用户未提及时 PRD 写「无 global-notice」，UI 不绘制 Notice。
5. **第 2 轮不可跳过**：home-page 必问 HA01（模块多选）；流程多步必问 FB01；答完才允许输出 PRD。
